编写中台图库置信度算法，多次改版验证之后在产品中加以应用
  • 项目背景

由于人工智能设计项目涉及智能匹配与机器识别的属性，数据库的支持就显得不可或缺。而在此具体项目之中，数据库的内涵是指带有全面标签信息的版权图片库。而由于技术与成本的限制，此数据库的建立大部分依赖人力完成。而如何架构一个完善的中台管理系统，快速响应前端的需求与保持后台的稳定性，成为了项目中需要解决的问题。而由于图片数据的标签需要人工标注，需要置信度算法以有效地验证标签是否可信。

中台图库打标签及管理系统部分高保真展示

  • 项目需求

以多个标签标注人员所打出的标签数据作为依据，利用算法进行处理，得到置信度结果，设定合理的阈值，超过阈值的置信度被认为是标签可信。

  • 算法研究
  • 本算法是在熵算法的基础之上进行改良，下主要对比两者进行介绍

主要从有效列算法、标签量加权以及数据后处理三个方面进行考量，最终实现了对各种情况的有效应对，弥补了基础的熵算法区分度低、指标重合、阈值难以选取的问题。

  • 算法实现

敲定算法以及进行数据测试之后，与开发部门配合，实现了对于打标签系统置信度算法样例的开发

def cal(st):
    n = col
    n1 = 0
    t = col * row
    t1 = 0

    p0 = [0] * col
    p = [0] * col
    r = [0] * col
    value = [0] * col

    # A:65 B:66 Q:81
    for i in range(66, 66 + col):

        dataRange = chr(i) + str(st + 1) + ':' + chr(i) + str(st + row)
        data = sheet.range(dataRange).value
        count = get_count_per_col(data)
        p0[i - 66] = count / row

        t1 += count
        if count > 0:
            n1 += 1

    if n1 == 0:
        return None

    for i in range(0, col):
        p[i] = p0[i] / n1

    H1 = 0
    
    for i in range(0, len(p)):
        if p[i] == 0:
            continue
        H1 += p[i] * math.log(p[i])
    H1 *= -1

    if H1 == 0:
        H1 = 0.001

    H2 = math.log(col)

    for i in range(0, len(p)):
        r[i] = p[i] * H2 / H1 * t1 / t
        value[i] = math.atan(r[i] * n) * 2 / math.pi
    
    return value


def get_count_per_col(arr):
    count = 0
    # print('getNum')
    for item in arr:
        if item == 1:
            count += 1
    return count

3、企业运行、管理及技术方面的认识及体会

企业运行与管理

由于实习所在公司为创业公司，故在运行与管理的方面都比较扁平化，没有大厂复杂的管理架构和运行模式，但又正由于公司规模较小，不同部门之间的管理及协作也相对清晰，作为初创阶段的公司也经常进行运行与管理方面的改革，故也能体会到一个新兴的高技术创业公司在管理与运行方面的思考与探索。以下是我结合自身观察与一些前辈的经验而得到的认识与体会。

  • 一切工作都是人的工作，管理与业务是分不开的

    很多时候在强调管理的重要性时经常会忽略业务在其中所起到的作用，尤其对于一个初创公司而言，公司的核心目标就是产出一个或者多个有价值、有市场竞争力的产品，而这时很重要的就是整个企业的效率。运行良好的标志就是有高效高质的产出，管理行为的目的也是如此。当管理与业务脱节的时候，对于管理者和被管理者都是极为不利的。管理与业务关联要紧密，这不仅指团队的领导者在业务方面需要有一定的了解与能力，也是指在管理时将业务与人联结起来，以业务作为管理的依据和目的。

  • 搭建领导梯队，作为教练、裁判，而不只是好的球员

    一个公司只有业务骨干是无法正常运作的，正如同一场足球比赛只有球员无法运作起来。对于创业公司而言，合伙人或是核心员工在业务能力上自然都有过人之处，但公司需要有人管理，也就是充当裁判和教练的角色。而需要做的是搭建领导的梯队，让业务骨干成为不同层级上的教练或是裁判，既不用专职管理，保持较高的效率，也可以发挥他们业务精良在管理上的优势。

  • 企业的本质永远是团队协作，任何个人都只是一份子

    事实就是现代企业之中，任何事务与项目都需要以团队协作的方式进行。能力再强，地位再高的个人也只是团队之中的一部分，无法脱离团队进行产出。而随着互联网的发展，协同的趋势还在进一步加强，这就意味着很多时候应该站在团队的视角上，不止是项目团队，开发团队，还可能是人事团队，管理团队，从团队的效率出发，以团队为单位进行管理，更加高效，也更有意义。

  • 优秀的人喜欢和优秀的人共事，优秀的人更享受产生价值的过程

    企业在招募员工时势必将优秀作为最重要的指标，但问题在于优秀的人往往成堆出现，这不止是优秀的人之间的相互吸引，也是优秀的人的带动作用。所以要想招募到优秀的员工，首先需要公司有一个优秀的氛围以及众多优秀的人。而优秀的人往往期望从事更有价值的工作，这个价值并非指工资或待遇，而是更加有意义的和战略性的，比如创造出最好的产品，改变人们对于一种事物的认知。真正优秀的人更加看重成就感与使命感，所以需要企业在文化与价值方面有吸引力。

一些实习期间的思考

  • 关于跨专业协作

由于公司项目的性质是人工智能加设计，所以在项目团队中，不仅会有计算机，人工智能领域的同学，还有完全设计背景的同学。而在这种有复杂的学科背景的团队之中，如何进行有效的团队协作是较为重要的问题，而本身作为交叉专业的学生，这种问题的回答更为重要。通过请教前辈以及在团队中的亲身感受，以下可能是跨专业协作中较为重要的几点：

  1. 不同专业语料之间的交流

    这点具体是指，不同专业都有属于自己的术语，这种术语对于非本专业的团队成员来说，往往是难以理解的。更有可能，不同专业相同的词可能表达不同的意思，这就造成了沟通与理解时的巨大成本。所以可能的解决方法是建立专属于此团队的语料库，用以解释不同专业的专业术语，团队成员不断完善以及利用此语料库，以减少不同成员之间的沟通成本。

  1. 对其他领域人才的信任

    之所以跨专业协作比较困难，是因为团队成员之间无法做到相互信任。拿产品开发的流程来说，产品经理无法信任设计师，对设计作品的美观性进行怀疑，而开发者又难以信任产品经理，怀疑其需求的合理性和必要性。实际上，跨专业合作的目的就是专业的人做专业的事，而这种目的能实现的前提就是不同专业人才的相互信任，能够相信别人在他们的专业领域的能力与眼光，才能高效无芥蒂的进行沟通与合作。

  1. 带着方案进行沟通

    事实上就是，跨专业之间的协作本身就有着巨大的沟通需求，沟通是处理协作效率不高这一问题的有效手段，而一种比较合理且高效的手段就是带着方案交流。可能会有不同成员对于同一问题之间的分歧，但是单纯靠言语的交流有时候是无力的，这时候带着方案交流就很有必要，两种方案的初步实现就可能看出两者的高下。而另一种可能的情况是，开发端无法满足需求或是设计好的原型，这时候带着替代的方案交流可能更加高效也更有意义。

  • 关于身份

难以否认的一点是，在实习过程中，是难以摆脱自己的身份的。这里所说的身份，是指清华大学智能工程与创意设计专业大二学生。而之所以试图摆脱，是因为尽管实习公司本身就是来自清北的创业团队，这种身份仍然是特殊的，这种特殊性不仅来自学校，也来自专业。

在寻找实习的阶段，我就曾不止一次地感受到这种身份的特殊性所带来的不同，多家公司愿意提供面试机会的原因我想是与学校有着非常大的关系，招聘网站打招呼时的那句“学弟好”就可以充分说明问题。也正因为如此，在进行每次面试时，我都异常地紧张，害怕自己表现糟糕而连累影响清华的形象。而除此之外，由于智能工程与创意设计专业是一个比较新的专业，我一般是面试官所面试的这一专业的第一位学生，如果无法有好的表现的话，也会影响企业对于本专业的印象。

而这种特殊性在我进入公司后仍然没有改变，更多转变成了同事对于学校与专业的好奇，而在分配任务时也更倾向于给我较为脑力相关的工作，比如编写，总结，创造，而非机械重复的非脑力劳动。这让我在实习的经历之中获得了诸多的便利。但却让我处于长久的怀疑之中，怀疑所分配的工作是根据我的能力还是我的身份，怀疑给我的赞扬是否是一种基于身份的鼓励，怀疑我是否能体会到真实的企业内部，还是说我和真实之间隔了一层身份带来的厚厚障壁。

在学校的时候，由于所面对的是老师与同学，很少能体会到身份所带来的差异性与隔阂，但这次企业实践像是一个口子，让我第一次较长久地置身于学校之外的环境，如何处理这层身份所带来的优待与压力，偏见与隔阂都是我需要思考的问题。现在的我还无法找到合适的应对方式，但“如果我不是清华学生，这件事会怎样？”这个问题，我今后会常常去想。
